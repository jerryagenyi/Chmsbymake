## ðŸ“Œ Manual Development Notes

### For Technical Writers:
- Each chapter should be 2-4 pages
- Include screenshots for every major action
- Use step-by-step numbered instructions
- Add "Tip" callout boxes for best practices
- Add "Warning" boxes for critical actions
- Include "See Also" cross-references
- End each chapter with summary checklist

### Screenshot Requirements:
- Desktop view (1920x1080)
- Mobile view (375x667)
- Annotated with arrows/highlights
- Consistent branding (hide test data)
- Dark theme primary, light theme secondary
- All screenshots numbered sequentially

### Style Guide:
- **Bold** for UI elements (buttons, menus)
- *Italics* for emphasis
- `Code font` for URLs, values, technical terms
- Clear, concise language (grade 8-10 reading level)
- Active voice ("Click the button" not "The button is clicked")
- Second person ("You can..." not "Users can...")

### Page Layout:
- A4 size (210mm x 297mm)
- 1-inch margins all sides
- Header: Page title + page number
- Footer: "ChurchAfrica ChMS Admin Manual v1.0"
- Two-column layout for dense sections
- Full-width for screenshots
