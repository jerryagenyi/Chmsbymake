## ðŸ“Œ Manual Development Notes

### For Technical Writers:
- Simple, friendly language (grade 6-8 reading level)
- Short paragraphs (3-4 sentences max)
- Numbered steps for all procedures
- Lots of screenshots (every major screen)
- Use emojis sparingly (only in headers)
- "You" language (second person)
- Active voice only

### Screenshot Requirements:
- Mobile screenshots (375x667 - iPhone SE size)
- Desktop screenshots (1366x768 - common laptop)
- Both iOS and Android where different
- Clear annotations (arrows, highlights, numbers)
- Callout boxes for important info
- Consistent branding

### Design Elements:
- Friendly, welcoming tone
- Colorful icons and illustrations
- "Tip" boxes with light bulb icon
- "Important" boxes with exclamation icon
- "Success" boxes with checkmark icon
- Step-by-step numbered boxes
- Before/after comparisons

### User Personas:
- **Primary:** Sarah, 35, regular member, moderate tech skills
- **Secondary:** Pastor John, 55, leader, low tech skills
- **Tertiary:** David, 22, youth member, high tech skills

### Page Layout:
- A4 size (simpler for printing)
- Large margins (easier to read)
- Page header: Chapter name
- Page footer: Page number + "Member Portal Guide"
- Single column layout (mobile-friendly)
- Large fonts (14pt body, 24pt headings)

### Accessibility:
- High contrast text
- Large font options
- Alternative text for all images
- Screen reader compatible
- Color-blind friendly (don't rely on color alone)
